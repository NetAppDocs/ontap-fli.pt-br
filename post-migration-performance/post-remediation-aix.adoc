Após a migração, os hosts AIX requerem etapas de correção para garantir que os LUNs recém-importados sejam reconhecidos e funcionais.



== Pós-migração - FLI offline

Para FLI offline, coloque o LUN online após excluir a importação do LUN e confirmar que os iniciadores do host estão conectados.

NOTE: Consulte a documentação do fornecedor do sistema operacional para obter a lista de comandos mais recente e atualizada.

.Passos
. Faça uma nova verificação após desconectar para confirmar que o estado está limpo.
+
[source, cli]
----
cfgmgr
----
. Identificar dispositivos de armazenamento.
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. Verifique se o dispositivo está conectado.
+
[source, cli]
----
lspv |grep hdiskx
----
. Identifique o grupo de volume e os sistemas de arquivos.
+
[source, cli]
----
lspv
lsvg <vg_name>
lsvg -l <vg_name>
lsvg -p <vs_name>
----
. Varionvg e montar sistemas de arquivos.
+
[source, cli]
----
varyonvg <vg_name>
mount /mount_point
df -g
----




== Pós-Migração - FLI Online

Após excluir a relação de importação, confirme se o novo LUN opera independentemente com UNMAP ativado. Após excluir a importação do LUN, verifique se o LUN está configurado com  `-space-reserve disabled` e  `-space-allocation enabled`. Caso contrário, modifique-o de acordo.
