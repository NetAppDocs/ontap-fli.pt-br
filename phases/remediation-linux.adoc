= Remediação de host Linux ONTAP FLI
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Antes de executar a migração ONTAP FLI em um host Linux, conclua estas etapas de pré-migração. Existem etapas separadas para migrações offline e online.



== Pré-migração - FLI offline

Esses passos são comuns tanto para FLI online quanto offline. Antes de remover o LUN de origem do host, certifique-se de que os seguintes passos sejam concluídos.

[cols="3"]
|===
| Sistema de arquivos (LVM e não-LVM) | Pacemaker com LVM (SCSI3 PR) | SANBOOT 


| Etapa 1: Coletar completamente a configuração do dispositivo de bloco de ponta a ponta: integridade/caminhos do multipath, layout LVM no dispositivo de bloco e configurações do GRUB/bootloader confirmando o dispositivo de inicialização correto e os parâmetros de inicialização corretos. | Etapa 1: Coletar completamente a configuração do dispositivo de bloco de ponta a ponta: integridade/caminhos do multipath, layout LVM no dispositivo de bloco e configurações do GRUB/bootloader confirmando o dispositivo de inicialização correto e os parâmetros de inicialização corretos. | Etapa 1: Coletar completamente a configuração de inicialização SAN de ponta a ponta: integridade/caminhos do multipath, layout LVM no LUN de inicialização e configurações do GRUB/bootloader confirmando o dispositivo de inicialização e os parâmetros de inicialização corretos. 


| Etapa 2: Certifique-se de que as  `/etc/fstab` entradas para os respectivos dispositivos de dados (LVM/Não-LVM) sejam referenciadas pelo seu UUID do sistema de arquivos. | Etapa 2: Desative os grupos de recursos do cluster para os respectivos LUNs que estão sendo migrados. Após desativá-los, exclua os grupos de recursos. | Etapa 2: Certifique-se de que as  `/etc/fstab` entradas para o dispositivo SAN Boot, incluindo os dispositivos de dados (lvm/nonlvm), sejam referenciadas pelo seu UUID do sistema de arquivos. 


| Etapa 3: Certifique-se de que todas as operações nos pontos de montagem de destino estejam pausadas. Certifique-se de que não há processos acessando o dispositivo de bloco. Se necessário, drene/encerre os handles de processos abertos. | Etapa 3: Execute o backup da configuração do LVM VG para um arquivo, referente aos respectivos grupos de recursos que acabamos de desativar/excluir. Em seguida, desative os VGs. | Passo 3: Certifique-se de ter atualizado a  `GRUB_CMDLINE_LINUX` linha com as referências UUID corretas. Recrie uma nova configuração do GRUB com base nas referências UUID do dispositivo. 


| Etapa 4: Desmonte o sistema de arquivos e execute o backup da configuração do grupo de volume LVM. Desative o VG e exporte o VG | Etapa 4: remova os LUNs de origem da configuração stonith do PCS. | Etapa 4: valide as alterações com as novas configurações do GRUB e recrie a nova imagem initrd para refletir as alterações no dispositivo de boot SAN. 


| Passo 5: limpe a configuração multipath do dispositivo e remova todos os arquivos de dispositivo SCSI obsoletos. | Passo 5: Limpe a configuração multipath do dispositivo e remova todos os arquivos de dispositivo SCSI obsoletos. Nota: se alguma reserva persistente de LUN for detectada, limpe-a e certifique-se de que não haja chaves PR. | Etapa 5: reinicie o host para garantir que ele inicialize corretamente após as alterações acima. Assim que o host inicializar corretamente, entenda que a pré-remediação do host para a migração do FLI está pronta. 


| Etapa 6: Recarregue a configuração multipath para que o host tenha a lista de dispositivos atualizada. Se necessário, execute uma nova verificação do barramento SCSI do host. | Etapa 6: analise novamente o host para garantir que não haja entradas obsoletas no mapeamento de dispositivos existente. | Etapa 6: Agora desligue o host sanboot para prosseguir com o fluxo de trabalho ONTAP FLI. 
|===


== Pré-migração - FLI online

A seguir estão os passos específicos do FLI online, mas estes passos devem ser precedidos pelos passos do FLI offline de pré-migração. Os passos a seguir devem ser executados após o LUN da NetApp ser provisionado para o host. Esses passos devem ser executados após o LUN da NetApp ser provisionado para o host. E isso se aplica somente ao FLI online.

[cols="3"]
|===
| Sistema de arquivos (LVM e não-LVM) | Pacemaker com LVM (SCSI3 PR) | SANBOOT 


| Etapa 1: execute uma nova verificação de armazenamento em todos os hosts Linux mapeados. | Etapa 1: execute uma nova verificação de armazenamento em todos os hosts Linux mapeados. | Passo 1: Ligue o servidor. 


| Etapa 2: Descubra os dispositivos multipath. | Etapa 2: descubra os dispositivos multipath. Certifique-se de que os LUNs migrados sejam adicionados à configuração do PCS Stonith. | Passo 2: No BIOS ou UEFI do HBA do servidor, configurar o novo dispositivo de inicialização que é apresentado a este HBA do host. 


| Etapa 3: Importe os grupos de volume e execute a ativação do VG. | Etapa 3: execute a restauração do backup de configuração do VG para os VG/LVM recém-descobertos, seguida pela ativação do VG. | Etapa 3: reinicie o host para inicializar a partir do LUN recém-mapeado que faz parte da Importação de LUN. 


| Passo 4: Monte a configuração LVM nos respectivos pontos de montagem. | Etapa 4: Crie os novos grupos de recursos PCS e adicione os respectivos recursos para os volumes lógicos (LV) e sistemas de arquivos disponíveis. NOTA: Observe que os objetos LVM recém-ativados mantêm as estruturas de LV e sistema de arquivos intactas. |  


|  | Etapa 5: Verifique o status do cluster PCS para garantir que todos os recursos estejam online e disponíveis. |  
|===
